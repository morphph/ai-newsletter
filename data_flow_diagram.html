System Components & Data Flow

Data Sources: External content providers
Collection: Content fetching services
Processing: Filtering & enrichment
Storage: Database layer
API: Backend services
Frontend: User interface
Scheduler: Automation trigger

Processing Pipeline

  1. Scheduling: GitHub Actions or manual trigger initiates the crawler
  2. Collection: News Crawler v3 orchestrates parallel fetching from websites (via Firecrawl) and Twitter
  3. Pre-filtering: Apply keyword/pattern matching to reduce processing costs
  4. AI Filtering: GPT-4 evaluates content for AI/ML relevance
  5. Summarization: Generate concise summaries for relevant content
  6. Storage: Save to Supabase with proper source relationships
  7. API Service: FastAPI exposes endpoints for frontend consumption
  8. Presentation: React frontend displays filtered, summarized content
