üîÑ Processing Pipeline Details

üì∞ Article Pipeline

  1. Homepage Scrape: Fetch homepage HTML via Firecrawl
  2. Extract Links: Parse all article URLs from homepage
  3. Date Filter: Python filters for yesterday's articles only
  4. GPT AI Filter: Batch evaluate titles/snippets for AI relevance
  5. Selective Scrape: Only scrape full content of AI articles
  6. Store: Save to articles table with AI flag
  7. Summarize: Generate summaries for stored articles

Efficiency: Reduces Firecrawl API calls by ~80% by only scraping AI-relevant articles

üê¶ Tweet Pipeline

  1. Fetch by Date: Twitter API query for yesterday's tweets
  2. GPT Batch Filter: Send all tweets to GPT for AI evaluation
  3. Store AI Tweets: Only store tweets identified as AI-related
  4. Batch Summarize: Group by author for context
  5. Extract Tags: Identify key AI topics mentioned

Optimization: Batch processing reduces GPT calls, only stores relevant tweets

üîë Key Optimizations

  • Different Filters: Articles use date + AI filters, Tweets use date-based API query + AI filter
  • Batch Processing: Both pipelines use batch GPT evaluation to reduce API calls
  • Selective Scraping: Articles are only fully scraped if AI-relevant
  • Pre-filtering: Date filtering happens before expensive operations
  • Parallel Processing: Articles and tweets are processed concurrently
